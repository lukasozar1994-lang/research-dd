# Research-Driven Setup

A framework installer for research-driven development in VS Code with GitHub Copilot Chat.

## What It Does

`research-driven-setup` converts a fresh or existing workspace into a fully configured environment for research-driven development. It generates:

- **Agent files** (`.github/agents/`) — Custom Copilot agents for deep research, planning, implementation, Playwright debugging, and test scenario generation.
- **Prompt files** (`.github/prompts/`) — Pre-built prompts for common workflows.
- **Skill files** (`.github/skills/`) — Domain-specific skill modules referenced by agents.
- **Instruction files** (`.github/instructions/`) — Coding guidelines for Python, Playwright, etc.
- **Scripts** (`.github/scripts/`) — MCP startup scripts and workflow automation.
- **MCP configuration** (`.vscode/mcp.json`) — Declarative MCP server configuration.
- **NPM dependencies** (`package.json`) — Local MCP server packages.

## Installation

### Quick Start (via uvx)

```bash
uvx --from git+https://github.com/lukasozar1994-lang/research-dd.git@v0.1.0 research-driven-setup
```

### From Local Checkout

```bash
git clone https://github.com/lukasozar1994-lang/research-dd.git
cd research-dd
uv tool install -e .
research-driven-setup
```

### Development Install

```bash
git clone https://github.com/lukasozar1994-lang/research-dd.git
cd research-dd
python3 -m venv .venv
source .venv/bin/activate
pip install -e .
research-driven-setup
```

## Usage

### Interactive Install

Run without arguments for an interactive guided installation:

```bash
research-driven-setup
```

The installer will:
1. Check environment prerequisites (Python, Node.js, npm, Git)
2. Let you select an installation profile
3. Inspect existing workspace state
4. Generate all workflow assets
5. Configure MCP servers
6. Prompt for optional secrets (Context7 API key)
7. Offer to set up GitHub MCP authentication
8. Install npm dependencies
9. Generate an install report

### Non-Interactive Install

```bash
research-driven-setup install --non-interactive
```

### Available Commands

| Command | Description |
|---------|-------------|
| `research-driven-setup` | Interactive install (default) |
| `research-driven-setup install` | Explicit install flow |
| `research-driven-setup doctor` | Validate workspace health |
| `research-driven-setup update` | Update to newer framework version |
| `research-driven-setup list-profiles` | List available profiles |
| `research-driven-setup report` | Show last install/doctor report |
| `research-driven-setup --help` | Show help |
| `research-driven-setup --version` | Show version |

### Profiles

| Profile | Description |
|---------|-------------|
| `ubuntu-research` (default) | Full research-driven workflow for Ubuntu |
| `ubuntu-research-playwright` | Adds Playwright browser automation |

## Customization

### Profiles

Profiles control which agents, prompts, skills, and MCP servers are installed. The default `ubuntu-research` profile includes all core workflow capabilities.

### Managed Files

Files generated by the installer follow a conflict policy:
- **skip** — Don't overwrite if the file already exists (most agents, prompts, skills)
- **overwrite** — Always update to the latest version (scripts, MCP config)
- **block** — Never touch user-owned files

### Secrets

Secrets are never committed to version-controlled files:
- `CONTEXT7_API_KEY` is stored in `.env` (which should be in `.gitignore`)
- `.env.example` provides guidance on required variables
- GitHub MCP authentication is handled through VS Code's OAuth flow

## Troubleshooting

### Common Issues

**`research-driven-setup` command not found**
```bash
# If installed via uv:
uv tool install -e .

# If installed via pip:
source .venv/bin/activate
pip install -e .
```

**npm install fails**
```bash
# Ensure Node.js and npm are installed
node --version  # Should be 18+
npm --version   # Should be 9+
npm install
```

**MCP servers don't start in VS Code**
1. Open Command Palette (Ctrl+Shift+P)
2. Type "MCP: List Servers"
3. Start individual servers
4. For GitHub MCP, complete the OAuth flow

**Context7 API key issues**
```bash
# Set the key in .env
echo "CONTEXT7_API_KEY=your-key-here" >> .env
# Or run doctor to check
research-driven-setup doctor
```

### Running Diagnostics

```bash
research-driven-setup doctor
```

This checks:
- Environment prerequisites
- Install marker and version
- MCP server configuration
- Generated asset completeness
- npm dependency installation status
